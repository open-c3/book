# 关于连接器

```
现在考虑一个问题，假如你是运维部门的负责人，某些情况下，你们可能会考虑购买第三方系统来做一些特定的事情，
比如：堡垒机，作业平台、资源管理平台，账单系统。

我们会发现，很多平台不适合独立使用，它需要和已有的平台进行联动。

举个简单的例子，比如堡垒机，登录的部分我们可以通过LDAP进行对接，
但是堡垒机中需要管理的机器、机器和用户的关系、机器的分组，这部分内容怎么和内部系统联动。

所以可能需要第三方系统进行定制化开发。

OPEN-C3为了避免在企业落地过程中需要定制化开发，定义了一个“连接器”的概念。
```

**设计理念：**

```
OPEN-C3自身要提供一个完整的功能，安装后即可使用。

在可能需要和企业内部系统进行联动的部分，比如“登录功能”、“资源列表”，“权限验证”，把这部分功能划分到“连接器”的部分。

OPEN-C3定义了连接器中的的每一个接口，只需要按照文档实现对应的接口，就可以通过连接器配置把对应的功能切到内部平台。
【我们认为企业内部对自己的平台和自己平台的开发语言比较熟悉，所以应该通过添加接口的方式来完成联动的动作。】

比如：企业通过自己的权限管理系统实现了OPEN-C3验证权限的接口，在OPEN-C3上把权限验证的地址指向企业内部的接口地址，
之后OPEN-C3的权限验证部分的请求就会转到企业的权限系统中，以后的权限管理就可以集中在企业已有的权限系统中。
```
